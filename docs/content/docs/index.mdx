---
title: Introduction
description: A transpiler that compiles a clean .actio.yml source into the verbose GitHub Actions YAML you'd otherwise hand-write.
---

Actio adds a handful of macro keywords on top of standard workflow YAML. They expand
at **build time** — nothing survives into the generated output, so there's no runtime
and no lock-in. Everything that isn't a macro is passthrough: a macro-free
`.actio.yml` is already a valid workflow.

Here's the smallest useful example. [`fragments`](/docs/macros/fragments) defines
reusable steps, `inject` splices them in, and ordinary steps pass straight through:

<CodeCompare>

```yaml title=".actio.yml"
fragments:
  setup:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - inject: setup
      - run: npm test
```

```yaml title="generated .yml"
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm test
```

</CodeCompare>

## Why

| Pain in raw YAML | Macro |
| --- | --- |
| Dynamic matrix needs a hand-built setup job that prints JSON to `$GITHUB_OUTPUT`, consumed downstream via `fromJSON()`, with escaping and empty-guard gotchas | [`dynamic-matrix`](/docs/macros/dynamic-matrix) |
| A native matrix fans out anonymous jobs, so nothing downstream can `needs:` one specific leg | [`expand_matrix`](/docs/macros/expand-matrix) |
| Reusing steps forces a separate [composite action](https://docs.github.com/en/actions/sharing-automations/creating-actions/creating-a-composite-action) or [reusable workflow](https://docs.github.com/en/actions/sharing-automations/reusing-workflows) file | [`fragments`](/docs/macros/fragments) |
| Retrying flaky steps requires bash loops or third-party actions | [`retry`](/docs/macros/retry) |
| try/catch means smearing `if: failure()` / `continue-on-error` across steps | [`fallback`](/docs/macros/fallback) |
| Compile-time constants and structured defaults are hard to keep typed in plain YAML | [`params`](/docs/macros/params) |

Prior art ([`github-actions-workflow-ts`](https://github.com/emmanuelnk/github-actions-workflow-ts),
[`github-actions-wac`](https://github.com/webiny/github-actions-wac),
[`projen`](https://github.com/projen/projen)) does typed object → YAML 1:1
serialization. Actio's differentiator is the **macro/transform compiler**.
