---
title: job_defaults
description: Inject shared default settings across every job at compile time.
---

Define default job-level settings once at the top level; Actio merges them into
every job at compile time and strips both `job_defaults` and `executors` from the
output — zero residual directives.

Both jobs inherit `runs-on` and `timeout-minutes`; `env` is deep-merged so `lint`
ends up with both `CI` and `DEBUG`:

<CodeCompare>

```yaml title=".actio.yml"
name: CI
on: [push]
job_defaults:
  runs-on: ubuntu-latest
  timeout-minutes: 30
  env:
    CI: "true"
jobs:
  test:
    steps:
      - run: npm test
  lint:
    env:
      DEBUG: "1"
    steps:
      - run: npm run lint
```

```yaml title="generated .yml"
jobs:
  test:
    runs-on: ubuntu-latest
    timeout-minutes: 30
    env:
      CI: "true"
    steps:
      - run: npm test
  lint:
    runs-on: ubuntu-latest
    timeout-minutes: 30
    env:
      CI: "true"
      DEBUG: "1"
    steps:
      - run: npm run lint
```

</CodeCompare>

## Merge semantics

| Key | Behaviour |
|---|---|
| `if` | **AND-combined** — `default && inline`. Never replaced. |
| `permissions`, `concurrency`, `strategy` | **Replace-on-presence** — inline value wins entirely; default only applies when the key is absent. |
| `runs-on`, `timeout-minutes`, `continue-on-error`, `environment` | **Inline wins** — job value takes precedence. |
| `env`, `container`, `services`, `defaults` | **Deep-merged** — job keys overlay the default. |

<CodeCompare>

```yaml title=".actio.yml"
name: CI
on: [push]
job_defaults:
  if: ${{ github.ref == 'refs/heads/main' }}
  env:
    REGION: us-east-1
jobs:
  deploy:
    runs-on: ubuntu-latest
    if: ${{ success() }}
    env:
      REGION: eu-west-1
    steps:
      - run: ./deploy.sh
```

```yaml title="generated .yml"
jobs:
  deploy:
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main' && success()
    env:
      REGION: eu-west-1
    steps:
      - run: ./deploy.sh
```

</CodeCompare>

<Callout title="Reusable-workflow call jobs">
  Jobs with `uses:` only receive the call-compatible subset of defaults: `if`,
  `permissions`, `concurrency`, and `strategy`. Runner keys like `runs-on`,
  `env`, and `timeout-minutes` are silently skipped (an `info` diagnostic is
  emitted listing what was dropped). `executor:` is not supported on call jobs.
</Callout>

## executors

Named runner presets — a companion to `job_defaults` for extracting repeated
runner configuration into a reusable template.

<CodeCompare>

```yaml title=".actio.yml"
name: CI
on: [push]
executors:
  hardened:
    runs-on: ubuntu-latest
    timeout-minutes: 20
    env:
      HARDENED: "true"
jobs:
  test:
    executor: hardened
    steps:
      - run: npm test
```

```yaml title="generated .yml"
jobs:
  test:
    runs-on: ubuntu-latest
    timeout-minutes: 20
    env:
      HARDENED: "true"
    steps:
      - run: npm test
```

</CodeCompare>

Compose multiple executors by passing a list — they are merged left-to-right,
with later entries overriding earlier ones for scalar keys:

```yaml title=".actio.yml"
executor: [base, gpu]
```

An inline runner key on the job always wins over what the executor provides.
