---
title: params
description: Typed compile-time inputs interpolated with double-brace tokens.
---

Define compile-time parameters once, then interpolate them with `{{ params.x }}`
for scalars or `{{ toJSON(params.x) }}` for objects/lists.

`params` is stripped and literals are fully resolved:

<Tabs items={['.actio.yml', 'generated .yml']}>
  <Tab value=".actio.yml">

  ```yaml
  name: Deploy
  on: [push]
  params:
    env:
      type: enum
      values: [dev, staging, prod]
      default: staging
    retries:
      type: number
      default: 3
  jobs:
    deploy:
      runs-on: ubuntu-latest
      steps:
        - run: echo "env={{ params.env }} retries={{ params.retries }}"
  ```

  </Tab>
  <Tab value="generated .yml">

  ```yaml
  jobs:
    deploy:
      runs-on: ubuntu-latest
      steps:
        - run: echo "env=staging retries=3"
  ```

  </Tab>
</Tabs>

## Token rules

- `{{ ... }}` is compile-time interpolation.
- `${{ ... }}` is GitHub runtime passthrough (except reserved roots handled by other macros).
- `${{ params.* }}` is invalid — `params` are compile-time-only.
